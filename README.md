# TDD における用語の意味

## 三角測量
同じ対象に対して違う観点のテストを追記して、プロダクトコードがきちんと動作するかどうかを測る  
そのテストがたまたま通過したということが起こらないように、状況によって、２つ以上の値を使用しテストを書く。

## レッド・グリーン・リファクタリング
レッド：テストに失敗するコードを書く  
グリーン：テストに成功するコードを書く  
リファクタリング：読みやすいコードへ改善する  

## 仮実装
テストに失敗した時にテスト自体がおかしいのか、実装したコードがおかしいのか判別できなくなる場合があるので、仮実装をして、テストを通過させる。
